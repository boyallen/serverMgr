# GM管理后台 PRD

## 1. 项目概述

### 1.1 项目背景
为游戏运营团队提供一个简单易用的服务器管理工具，用于快速执行常用服务器操作。

### 1.2 项目目标
- 提供直观的服务器状态监控
- 简化服务器启停操作流程
- 支持定时命令执行
   - 提高运营团队工作效率

## 3. 技术规格
详见 Server-Manage-TRD.md

## 4. 页面设计

### 4.1 主控制台
- **左侧菜单**：服务器管理

### 4.2 服务器管理页面
- 居中显示
- 服务器列表表格
- 每行包含：服务器信息、状态指示器、操作按钮组
#### 4.2.0 获取服务器列表
- 页面加载时自动调用获取服务器列表接口
- 根据接口返回的数据动态渲染服务器列表

#### 4.2.1 服务器列表表格
- 列表字段应与接口返回数据一致：
  - **服务器ID**：直接显示
  - **服务器名称**：根据当前语言环境显示对应的名称
  - **服务器地址**：显示网关地址 (GateAddr)
- **操作按钮组**：
  - 启动 (绿色)
  - 停止 (红色)
  - 重启 (黄色)
  - 设置时间 (蓝色)
- **样式要求**：
  - 字体大小：16px
  - 内容居中显示

##### 4.2.1.1 服务器信息
- 展示服务器ID和名称
- 支持多语言名称展示

##### 4.2.1.2 设置时间
- 设置时间：选择时间 可以提供一个日期选择器
- 确认按钮：确认设置
- 取消按钮：取消设置
##### 4.2.1.3 操作按钮组
   - 几个按钮需要并排
   - 启动按钮：绿色
   - 停止按钮：红色
   - 重启按钮：黄色
   - 设置时间按钮：蓝色
   - 按钮居中：居中

## 4. 交互设计

### 5.1 操作流程
1. 执行操作 → 显示结果反馈

### 5.2 反馈机制
- 操作成功：绿色提示消息
- 操作失败：红色错误提示
- 操作确认：模态对话框
- 加载状态：进度条/加载动画

## 5. 安全考虑

### 6.1 安全措施
- 敏感操作需要二次确认
- 会话超时自动登出
- 防止CSRF攻击

## 6. 部署要求

### 7.1 环境要求
- Node.js 16+
- 现代浏览器支持
- 手机浏览器兼容
- HTTPS协议访问

### 7.2 性能要求
- 页面加载时间 < 3秒
- 操作响应时间 < 1秒
- 支持并发用户数 > 10
